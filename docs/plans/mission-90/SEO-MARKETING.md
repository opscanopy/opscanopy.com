## 2. The binding loop (one progress store, three readers)
`oc-m90-v1` — schema & parsing owned by `src/lib/mission90/progress.ts` (pure, tested); actual localStorage reads/writes live in page scripts (repo pattern). **Roadmap = the map, day page = the work, mission = the reward.** Every page ends by handing the user exactly one next action.
- **Daily loop:** hub resume card ("Continue → Day 13 · Phase 2 · ~50 min") → day page phase strip orients → completion moment (check anim, tomorrow teaser expands, next CTA promotes) → return tomorrow.
- **Weekly loop:** a mission day's page links the mission directly as its lab (primary path); its completion banner is the unlock-framing fallback ("Mission unlocked: Server Down! You have the skills now. → Play (15–20 min)") → victory screen stats + rank + copy-share debrief → "Week 1 conquered. Day 8 sharpens your text-fu →" hands back to content (next live day; draft → roadmap, never a dead end). Victory writes stats → roadmap mission badge fills, hub gains trophy count.
- **Discovery loop (SEO lands on day 24):** page answers the query standalone (the SEO contract — never gate/redirect); slim dismissible orientation banner (only when no stored state and day>1); "Builds on Day 23" link; cold phase strip shows position not progress; if they complete day 24 first, respect it (hub offers Day 25 + secondary "Start Phase 1" link).
- **Cold starts:** hub hero CTA = "Start Day 0: Setup (20 min)" (not Day 1 — the time promise survives first contact); Mission 1 card = "Playable now — no setup, runs in your browser" (**availability canon: live missions are always playable; `unlockAfterDay` is badge copy only** — the game is ALSO top-of-funnel, playable before reading anything); private-mode/cleared storage degrades to cold-start gracefully + "progress saved in this browser only" note.

## 3. SEO
**Phase → keyword clusters:** P1 "linux {topic} explained for devops" · P2 "{tool} {concept} tutorial", "docker/gha interview questions" · P3 "aws {service} for devops", "X vs Y" · P4 "kubernetes {concept} explained", "terraform state explained simply" · P5 "devops interview questions {n} years experience", "devops portfolio projects".
**Formulas:** title `{Primary keyword}: {angle} — Mission 90, Day {N}` (keyword first, ≤60 chars, drop suffix when tight; suffix deliberately avoids "90 Days of DevOps" — that's Michael Cade's established brand and would sabotage SERP differentiation); description `Learn {topic} in under an hour: {2–3 specifics}. Hands-on lab, the real errors I hit, and {k} real interview questions. Day {N} of the free 90-day DevOps path.` — the "under an hour / free / real errors" trio is the SERP differentiator.
**Headings:** only 2 keyword-carrying H2s (concept heading is topic-specific — never "Concept Notes"; Q&A heading = "{topic} Interview Questions"); the rest stay fixed template labels (brand asset, not spam).
**JSON-LD:** hub `courseLd` (Course + hasCourseInstance: online, free, `courseWorkload` = TOTAL program hours computed from the registry (≈PT79H) — per-day PT1H would be factually wrong structured data; syllabusSections from phases) + hub FAQ `faqPageLd`; days `techArticleLd` **with Person author + `isPartOf` → the Course entity** (requires extending the existing helper — it currently hardcodes Organization at jsonld.ts:109; covered in PLAN's jsonld.ts Modify entry) + `faqPageLd` from interviewQA + `breadcrumbLd`; missions `softwareAppLd` + breadcrumb. **Honest expectation:** post-2023 Google FAQ rich results are effectively gov/health-only — we ship faqPageLd for Bing + AI answer engines (real 2026 channel), not for Google snippets. On-page Q&A text is the asset regardless.
**Internal mesh:** prev/next as real `<a>` (crawlable chain); day → guide deep-links in Go Deeper; **edit the 7 existing guides** to add "This is Day N–M of Mission 90" callouts (authority pass-through from the site's strongest pages); day → tools: d~4 regex-log-tester · d~6/19 cron tools · d11–12 subnet-calculator/cidr-checker · d24/27 docker-run-to-compose · d32–34 gha-validator + expression-tester · d68+ k8s-resource-calculator · observability Go-Deepers → promql-explainer/alertmanager-route-tester (use `relatedTools`-style framing: "practice with"). Tools ↔ learners is the two-way moat.
**Hygiene:** only authored days build (no thin stubs — 7 great days beat 90 outlines); draft rows are text, not links; canonical always clean (UTMs on share links only); `npm run indexnow` is a **manual** post-deploy step (deploy does not chain it — it's in the T15 ship gate; run it after every day-publish deploy). **Don't:** keyword-stuff titles; cannibalize guides (day targets long-tail "explained/day-N", guide keeps the head term; day links guide for depth).

## 5. Hub IA + overwhelm mitigations (Persona A protection)
Order: resume card → hero (3 trust markers in 5s: free · no signup · 45–60 min/day) → 3-layer time model (answers "do I have time?" first) → phase journey → **missions teaser above the table** (the differentiator must not be buried) → 90-day accordion (staircase, not cliff) → FAQ ("Is it really free?" / "I only have 30 min some days" / "Do I need Linux?" (WSL2) / "What if I miss days?" (no-guilt, in writing) / "Will this get me a job?") → final CTA + founding note.
Day-page 5-second contract: Where am I? (phase strip + breadcrumb) · How long? (~50 min core · +30 optional) · What do I get? (3 goals box on top).
Mitigations: accordions kill the 90-row wall; time-tagged H2s + collapsed Go Deeper de-bulk the 9-part anatomy; streaks stay out of MVP guilt paths (no "missed/behind" state anywhere, no-guilt FAQ in writing); Day 0 keeps Day 1 inside the 60-min promise (trust is won or lost there); mission list shows 1 playable + 2 named "in production" + rest collapsed as "the road ahead" (honest, not vaporware).
